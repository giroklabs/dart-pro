# 📝 Firebase 보안 및 서비스 복구 가이드

## 1. 사고 개요
- **현상**: iOS용 `GoogleService-Info.plist` 파일이 깃허브 공개 저장소에 노출됨.
- **리스크**: 악의적인 사용자가 API 키를 탈취하여 프로젝트 자원(Auth, Firestore 등)에 무단 접근할 가능성 발생.

## 2. 원인 분석
- **파일 관리 미흡**: `.gitignore`에 등록되어 있었으나, 이미 깃 인덱스(Tracking)에 포함된 상태에서 푸시되어 설정 파일이 그대로 노출됨.

## 3. 해결 과정 (Step-by-Step)

### [Web 영역]
1. **보안 아키텍처 변경**: 클라이언트에서 키를 직접 가지지 않고, 서버의 `/api/config` 엔드포인트를 통해 동적으로 받아오도록 수정 완료.
2. **환경변수(.env) 관리**: 실제 API 키는 서버의 환경변수로만 관리하며, 소스코드 관리 대상에서 완전 제외.

### [iOS 영역]
1. **키 로테이션 (Rotation)**: 유출된 기존 키를 파이어베이스 콘솔에서 삭제하고, 새로운 키가 포함된 `GoogleService-Info.plist`를 발급받아 Xcode에 반영.
2. **깃 추적 강제 제외**: 로컬 파일은 유지하되 깃허브 저장소에서만 삭제하기 위해 다음 명령어 실행:
   ```bash
   git rm --cached "ios/dart pro/GoogleService-Info.plist"
   ```
3. **원격 저장소 정리**: 인덱스 삭제 후 커밋 및 푸시를 통해 깃허브에서 해당 파일을 완전히 제거.

## 4. 향후 재발 방지 대책
- **민감 정보 하드코딩 금지**: 어떠한 경우에도 소스코드(.js, .swift, .html)에 API 키나 비밀번호를 직접 적지 않음.
- **.gitignore 선제적 적용**: 새로운 파일을 만들기 전에 반드시 .gitignore에 등록되어 있는지 확인.
- **주기적 키 점검**: 유출 여부와 관계없이 주기적으로 API 키를 교체하여 보안성 유지.
